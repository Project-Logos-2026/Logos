## 6. Canonical Dev Script Location Policy

```
DEV SCRIPT CANONICAL LOCATION POLICY
Version: 1.0
Effective: 2026-03-11
```

### Rules

**Rule 1 — Root Directory**
All development tooling scripts must live under:
```
/workspaces/Logos/_Dev_Resources/_Dev_Scripts/
```
Runtime modules must NOT be stored in this directory. Scripts that reside in
`LOGOS_SYSTEM/`, `logos/`, or any protocol core directory are runtime modules
and are out of scope.

**Rule 2 — Canonical Subdirectory Assignment**
All scripts must be placed in exactly one of the following canonical subdirectories,
determined by functional category:

| Functional Category | Canonical Directory |
|---------------------|---------------------|
| Repo scanning, auditing, triage | `ARCH_PRIME/` or `Repo_Tools/Repo_Audit/` |
| Static AST and semantic analysis | `ARCH_PRIME/` or `Repo_Tools/Static_Analysis/` |
| Dependency and import graph analysis | `ARCH_PRIME/` or `Repo_Tools/Dependency_Analysis/` |
| Canonical Import Facade tools | `Import_Facade/` |
| Phase 1–5 canonical pipeline | `P1-5/` (locked — no new files without approval) |
| Runtime Graph Engine audits | `RGE/` |
| File migration and namespace relocation | `Repo_Tools/Migration/` |
| Logic extraction utilities | `Repo_Tools/Code_Extraction/` |
| Architecture boundary validation | `Repo_Tools/Architecture_Validation/` |
| Runtime diagnostics and smoke tests | `Repo_Tools/Runtime_Diagnostics/` |
| Report and registry writers | `Repo_Tools/Report_Generation/` |
| Auto-generated lists, fixtures, inits | `Repo_Tools/Dev_Utilities/` |

**Rule 3 — Directory Naming Convention**
All directory names must follow `Title_Case_With_Underscores`. No lowercase,
no hyphens, no spaces. Examples: `Repo_Audit`, `Static_Analysis`, `Dev_Utilities`.

**Rule 4 — P1-5 Lock**
The `P1-5/` directory is a canonical phase-scoped directory. No scripts may be
added to or removed from `P1-5/` without explicit human approval. Auto-generated
files must not be placed in `P1-5/`.

**Rule 5 — Tool Index Maintenance**
After any new script is added or moved, update:

```
/workspaces/Logos/_Dev_Resources/_Dev_Scripts/Tool_Index/dev_scripts_inventory.json
/workspaces/Logos/_Dev_Resources/_Dev_Scripts/Tool_Index/dev_tool_registry.json
```

The Tool_Index must remain the single source of truth for tool metadata. Agents
resolving scripts for automated invocation must read from `Tool_Index/`, not from
filesystem discovery.

**Rule 6 — No Script Deletion**
No development scripts may be deleted without explicit human authorization, a
governance artifact update, and an entry in the session audit log.

**Rule 7 — Session Enforcement**
All sessions operating in this workspace that create, move, or modify dev tooling
scripts must enforce this policy. A session that cannot determine the correct
canonical location must write a `_pending_classification.txt` file in the same
directory as the script before committing.
