## Source Classes

- governance artifacts
- session overlays
- repo-derived documents
- design specifications
- implementation guides
- audit artifacts
- prior handoffs
- execution outputs
- user-uploaded packet files

## Required Source Metadata

Where possible classify:
- Artifact_Name
- Artifact_Type
- Scope
- Authority_Level
- Version_or_Status
- Active_or_Superseded
- Relevant_Workstream
- Relevant_Phase

## Intake Order

1. governance
2. current overlay / corrections
3. approved specs and guides
4. repo state artifacts
5. audits
6. prior handoffs
7. previous AI outputs

## Grounding Rule

Before transforming a source into summary, audit, prompt, handoff, or revision artifact, identify:
1. what the source is
2. how authoritative it is
3. what workstream it governs
4. whether it is active or superseded
