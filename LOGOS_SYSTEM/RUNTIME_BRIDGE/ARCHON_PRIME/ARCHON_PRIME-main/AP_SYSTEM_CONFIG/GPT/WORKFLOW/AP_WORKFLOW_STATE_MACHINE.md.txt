# AP WORKFLOW STATE MACHINE

This artifact defines the operational states of the ARCHON PRIME workflow pipeline.

The state machine ensures that development proceeds in a deterministic sequence.

Only actions allowed for the current state may be performed.

---

# STATE LIST

STATE_01_BRAINSTORM

Architect and GPT explore design ideas.

Allowed actions:

• brainstorming  
• conceptual analysis  
• architectural discussion  

Disallowed actions:

• prompt engineering  
• repository execution  

---

STATE_02_SPECIFICATION

Claude generates design specifications.

Allowed actions:

• spec drafting  
• design formalization  

Disallowed actions:

• prompt engineering  
• execution prompts  

---

STATE_03_PROMPT_ENGINEERING

GPT converts specifications into executable prompts.

Allowed actions:

• prompt generation  
• prompt validation  
• prompt schema compliance  

Disallowed actions:

• repository execution  

---

STATE_04_EXECUTION

VS Code executes prompts.

Allowed actions:

• repository modification  
• artifact generation  
• test execution  

Disallowed actions:

• architecture redesign  

---

STATE_05_RESULT_AUDIT

GPT evaluates execution results.

Allowed actions:

• execution analysis  
• validation checks  
• corrective prompt generation  

---

STATE_06_ITERATION

Workflow loops back to previous stages if necessary.

---

STATE_07_COMPLETE

Task is complete.

No further actions required.

---

# STATE TRANSITIONS


BRAINSTORM
→ SPECIFICATION
→ PROMPT_ENGINEERING
→ EXECUTION
→ RESULT_AUDIT
→ ITERATION or COMPLETE


---

# PURPOSE

The state machine prevents workflow drift and ensures:

• prompt generation occurs only after specifications  
• execution occurs only after prompts  
• auditing occurs after execution
