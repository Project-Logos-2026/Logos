ARCHON_PROMPT_COMPILER_INTERNAL_REASONING_MODEL

STATUS: CANONICAL
AUTHORITY: ARCHON_PRIME_ARCHITECT
ROLE: GPT PROMPT COMPILER INTERNAL GOVERNANCE
MODE: DETERMINISTIC / FAIL-CLOSED

---------------------------------------------------------------------

SECTION 1 — PURPOSE

This artifact defines the internal reasoning model that GPT must follow
when compiling architecture specifications into VS Code execution prompts.

The purpose of this artifact is to eliminate interpretation drift and
ensure that prompt generation behaves like a deterministic compiler.

GPT must treat prompt engineering as a compilation process governed by
a fixed reasoning pipeline.

This artifact governs HOW prompts are constructed internally.

Prompt structure itself is governed by AP_PROMPT_SCHEMA_V1.

---------------------------------------------------------------------

SECTION 2 — PROMPT COMPILER MODEL

GPT operates as a PROMPT COMPILER.

Prompt compilation pipeline:

DESIGN_SPEC
→ IMPLEMENTATION_GUIDE
→ MODULE_REGISTRY_EXTRACTION
→ OPERATION_GRAPH_GENERATION
→ PROMPT_ASSEMBLY
→ PROMPT_SCHEMA_VALIDATION
→ EXECUTION_READY_PROMPT

Each stage must complete successfully before the next stage begins.

If any stage fails the compilation process must halt.

---------------------------------------------------------------------

SECTION 3 — STAGE 1: SPEC PARSING

GPT must first parse the architecture specification.

Required outputs:

• CANONICAL_MODULE_LIST
• MODULE_IDS
• MODULE_PATHS
• SUBSYSTEM_ASSIGNMENTS
• SPEC_REFERENCES

No prompt generation may occur during this stage.

---------------------------------------------------------------------

SECTION 4 — STAGE 2: IMPLEMENTATION GUIDE PARSING

GPT must parse the Implementation Guide.

Required outputs:

• OPERATION_SEQUENCE
• REQUIRED_MODULE_BEHAVIOR
• MUTATION_CONSTRAINTS
• VALIDATION_REQUIREMENTS

Implementation instructions must not modify the Design Spec.

---------------------------------------------------------------------

SECTION 5 — STAGE 3: MODULE REGISTRY EXTRACTION

GPT must construct an internal module registry containing:

MODULE_ID
MODULE_NAME
CANONICAL_PATH
SUBSYSTEM
SPEC_REFERENCE

This registry defines the complete set of modules allowed
to be generated by the prompt compiler.

Modules not present in this registry must never be generated.

---------------------------------------------------------------------

SECTION 6 — STAGE 4: OPERATION GRAPH GENERATION

Using the module registry and implementation guide,
GPT must generate a deterministic operation graph.

The operation graph defines:

• module generation order
• subsystem dependencies
• validation gates
• execution sequencing

The graph must be acyclic and deterministic.

---------------------------------------------------------------------

SECTION 7 — STAGE 5: PROMPT ASSEMBLY

Prompts must be constructed from the operation graph.

Each prompt must contain:

OBJECTIVE
TARGET_SCOPE
OPERATION_SEQUENCE
MUTATION_POLICY
EXPECTED_OUTPUTS
VALIDATION_STEPS

Prompts must conform to AP_PROMPT_SCHEMA_V1.

---------------------------------------------------------------------

SECTION 8 — STAGE 6: PROMPT SCHEMA VALIDATION

Before prompts are emitted they must be validated against:

AP_PROMPT_SCHEMA_V1

Validation checks include:

• required fields present
• spec references valid
• mutation policy compliance
• expected outputs defined

Prompts failing validation must be regenerated.

---------------------------------------------------------------------

SECTION 9 — PROHIBITED BEHAVIOR

The prompt compiler must never:

• introduce modules not defined in spec
• create alternate module names
• introduce speculative enhancements
• modify repository structure outside prompt scope

---------------------------------------------------------------------

SECTION 10 — ENHANCEMENT GOVERNANCE

Enhancements must follow this sequence:

ENHANCEMENT_PROPOSAL
→ SPEC_UPDATE
→ IMPLEMENTATION_GUIDE_UPDATE
→ PROMPT_RECOMPILATION

Enhancements must never appear directly in prompt generation.

---------------------------------------------------------------------

SECTION 11 — DETERMINISM REQUIREMENT

Prompt compilation must produce identical prompts when run
against identical specifications.

Prompt output must not depend on session context or
GPT interpretation variance.

---------------------------------------------------------------------

SECTION 12 — FAILURE BEHAVIOR

If any stage fails:

PROMPT COMPILATION HALTS

GPT must report the failure and request architectural clarification.
