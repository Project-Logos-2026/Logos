## Purpose

This directory contains data artifacts that GPT must reference when engineering prompts. These are not instruction layers — they are context sources.

---

## Expected Contents

| Artifact | Role | Format |
|----------|------|--------|
| `repo_directory_tree.json` | Canonical path authority — verify file paths and module hierarchy | JSON |
| `repo_imports.json` | Import dependency map (when available) | JSON |
| `repo_python_files.json` | Python file inventory (when available) | JSON |
| `repo_symbol_imports.json` | Symbol-level import map (when available) | JSON |
| `Deep_Import_Violations.json` | Import violation dataset for remediation prompts (when available) | JSON |

---

## Usage Rules

1. GPT must reference `repo_directory_tree.json` to verify canonical file paths before including them in prompts
2. Import violation data drives import-facade remediation prompts
3. These artifacts are updated as the repo evolves — always use the most recent version available
4. If a context source is missing, GPT must note the absence and work with available information

---

## Note

These files are populated from the LOGOS repository and are not generated by the ARCHON_PRIME configuration system. They must be provided by the Architect or extracted from the repo.
