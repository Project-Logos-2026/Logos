# AP WORKFLOW PROMPT ENVELOPE SCHEMA

This artifact defines the standardized prompt envelope used in the ARCHON PRIME workflow.

The envelope separates **workflow orchestration metadata** from **execution content**.

This prevents workflow metadata from contaminating repository code or execution prompts.

---

# CONTROL PLANE VS DATA PLANE

All prompts exchanged between workflow participants are divided into two sections.

CONTROL PLANE  
Workflow metadata used for orchestration.

DATA PLANE  
Execution instructions consumed by the target environment.

The execution layer must only consume the DATA PLANE section.

---

# PROMPT ENVELOPE FORMAT

All prompts must follow the structure below.
=== AP_WORKFLOW_METADATA ===
STATE:
TARGET_PLATFORM:
PROMPT_ID:
TRACE:
ORIGIN:
TIMESTAMP:
=== END_AP_WORKFLOW_METADATA ===

=== EXECUTION_PROMPT ===
<execution instructions>
=== END_EXECUTION_PROMPT ===


---

# FIELD DEFINITIONS

STATE

Defines the current workflow state.

Example:


STATE: PROMPT_ENGINEERING


---

TARGET_PLATFORM

Defines the intended execution environment.

Allowed values:


GPT
CLAUDE
VSCODE


---

PROMPT_ID

Unique identifier for the prompt.

Example:


PROMPT_MOD_021


---

TRACE

Spec traceability reference.

Example:


SPEC-004 §3.2


---

ORIGIN

Defines the component that generated the prompt.

Example values:


GPT_PROMPT_COMPILER
CLAUDE_SPEC_ENGINE


---

TIMESTAMP

UTC timestamp indicating prompt generation time.

Example:


2026-03-09T18:10Z


---

# CRITICAL RULE

The metadata block is **workflow control information only**.

It must never be:

• written to repository code  
• inserted into generated artifacts  
• interpreted as implementation instructions  

The VS Code packet contains the interpreter responsible for removing metadata before execution.

---

# PURPOSE

The prompt envelope provides:

• deterministic workflow routing  
• prompt traceability  
• pipeline state awareness  
• debugging capability  

without affecting implementation artifacts.
